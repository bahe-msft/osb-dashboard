# Live-cluster end-to-end tests

The E2E suite starts a real dashboard server, connects it to an isolated
OpenSandbox cluster, and drives Microsoft Edge through `playwright-cli`.
It creates one sandbox, validates the live features, and deletes the sandbox at
the end. A best-effort cleanup also runs when a test fails.

## Requirements

- Go
- `curl`
- `playwright-cli`
- Microsoft Edge
- A kubeconfig with access to an isolated cluster where OpenSandbox is deployed

## Run

```bash
OSB_E2E_KUBECONFIG=/path/to/kubeconfig ./e2e/run.sh
```

Or with Just:

```bash
just e2e /path/to/kubeconfig
```

The following variables are optional:

| Variable | Default | Purpose |
| --- | --- | --- |
| `OSB_E2E_SANDBOX_IMAGE` | `python:3.12-slim` | Image created by the lifecycle test |
| `OSB_E2E_OPENSANDBOX_NAMESPACE` | `opensandbox-system` | Lifecycle service namespace |
| `OSB_E2E_SANDBOX_NAMESPACE` | `opensandbox` | Sandbox workload namespace |
| `OSB_E2E_PORT` | Random high port | Local dashboard port |
| `OSB_E2E_VIEWPORT_WIDTH` | `1600` | Browser viewport width |
| `OSB_E2E_VIEWPORT_HEIGHT` | `1000` | Browser viewport height (default is 16:10) |
| `OSB_E2E_RECORD` | `1` | Set to `0` to disable trace and video recording |
| `OSB_E2E_RUN_ID` | UTC timestamp and PID | Artifact directory name |

## Coverage categories

- **Application shell**: health endpoint and HTMX overview rendering
- **Navigation and preferences**: stable sidebar collapse and persisted theme
- **Sandbox lifecycle**: create a sandbox and delete it through the dashboard
- **Sandbox details**: running state, property layout, and live stats refresh
- **Terminal**: WebSocket connection, command execution, and connection
  preservation while switching between Details and Stats

The suite currently contains eight tests split into ordered category files under
`e2e/tests/`. Tests are intentionally serial because later categories operate on
the sandbox created by the lifecycle category.

## Debug recordings

Each run writes to:

```text
.playwright/e2e/<run-id>/
```

The directory includes:

- `server.log` — dashboard and upstream request logs
- `suite.log` — combined category results
- `<category>.log` — output from each independently executable suite file
- `run.webm` — recorded browser session, automatically scaled by Playwright to fit within 800×800 while preserving the viewport ratio
- Playwright trace output generated by `tracing-stop`
- `console.log` and `network.log`
- category screenshots and a failure screenshot when applicable

All generated artifacts stay under `.playwright/` and are ignored by Git.
